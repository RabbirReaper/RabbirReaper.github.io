SIC 介紹

Memory

8-bit bytes（8位元組）

  • 每個 byte（位元組）由 8 個 bits 組成
  • 這是最基本的記憶體單位
  • 可以儲存 0-255 的數值（2⁸ = 256 種可能）

3 consecutive bytes form a word（3個連續位元組構成一個字組）

  • 在 SIC 系統中，一個 word（字組） = 3 bytes = 24 bits
  • 這是 SIC 處理資料的基本單位
  • 大多數指令和運算都是以 word 為單位進行

32,768 bytes（32,768 位元組的記憶體容量）

  • 這是整個電腦記憶體的大小
  • 32,768 = 2¹⁵ = 32 KB
  • 記憶體位址範圍：0 到 32,767（十六進位：0000 到 7FFF）

Register

SIC 有 5 個暫存器

A (Accumulator，累加器)

  • 大小：24 bits (1 word)
  • 用途：用於算術和邏輯運算的主要暫存器
  • 功能：儲存運算結果、中間值

X (Index register，索引暫存器)

  • 大小：24 bits (1 word)
  • 用途：用於定址模式中的索引計算
  • 功能：幫助存取陣列或連續記憶體位置

L (Linkage register，連結暫存器)

  • 大小：24 bits (1 word)
  • 用途：儲存返回位址
  • 功能：呼叫副程式（subroutine）時，保存下一個指令的位址

PC (Program Counter，程式計數器)

  • 大小：24 bits (1 word)
  • 用途：指向下一個要執行的指令位址
  • 功能：自動遞增，控制程式執行流程

SW (Status Word，狀態字組)

  • 大小：24 bits (1 word)
  • 用途：包含各種狀態資訊
  • 功能：儲存條件碼（condition code）、中斷狀態等

Data Format

  • 範圍：-2²³ 到 2²³ - 1

  • 數值：-8,388,608 到 8,388,607

  • 說明：最高位元（bit 23）作為符號位

    0 表示正數
    1 表示負數

  • 無符號整數（Unsigned Integer）
    如果不考慮正負號：

範圍：0 到 2²⁴ - 1
數值：0 到 16,777,215

Instruction Formats（指令格式）

SIC 只有一種指令格式

  • 總長度：24 bits = 1 word = 3 bytes
  • opcode（操作碼）：8 bits，指定要執行的操作
  • x（索引位元）：1 bit
    • x = 0：直接定址
    • x = 1：索引定址
  • address（位址欄位）：15 bits，指定記憶體位址（0-32767）

Addressing Modes（定址模式）

SIC 有兩種定址模式：

Mode 1: Direct Addressing（直接定址）

  • 條件：x = 0
  • 計算：Target Address (TA) = address
  • 範例：LDA 1000
    • 從記憶體位址 1000 載入資料到 A

Mode 2: Indexed Addressing（索引定址）

  • 條件：x = 1
  • 計算：Target Address (TA) = address + (X)
  • 說明：(X) 表示暫存器 X 的內容
  • 範例：LDA 1000,X（假設 X = 50）
    • TA = 1000 + 50 = 1050
    • 從記憶體位址 1050 載入資料到 A
  • 用途：處理陣列、表格等連續資料結構

Instruction Set（指令集）

SIC 的指令可分為幾類：

A. 資料傳送指令（Data Transfer）

指令 說明
LDA Load A：載入資料到 A
LDX Load X：載入資料到 X
LDL Load L：載入資料到 L
STA Store A：儲存 A 到記憶體
STX Store X：儲存 X 到記憶體
STL Store L：儲存 L 到記憶體

B. 算術運算指令（Arithmetic）

指令 說明
ADD A = A + m：加法
SUB A = A - m：減法
MUL A = A × m：乘法
DIV A = A ÷ m：除法

註：m 表示記憶體中的值

C. 比較指令（Comparison）

指令 說明
COMP 比較 A 和 m，設定條件碼（SW）

結果存在 SW 中：

  • A < m：設為 <
  • A = m：設為 =
  • A > m：設為 >

D. 跳躍指令（Jump/Branch）

指令 說明
J Jump：無條件跳躍
JLT Jump if Less Than：A < m 時跳躍
JEQ Jump if Equal：A = m 時跳躍
JGT Jump if Greater Than：A > m 時跳躍

E. 副程式相關（Subroutine）

指令 說明
JSUB Jump to Subroutine：呼叫副程式
將返回位址存到 L，跳到指定位址
RSUB Return from Subroutine：返回
跳回 L 中儲存的位址

F. I/O 指令（Input/Output）

指令 說明
RD Read：從裝置讀取一個 byte 到 A
WD Write：將 A 的最右邊 byte 寫到裝置
TD Test Device：測試裝置是否就緒

G. 其他指令

指令 說明
TIX Test Index and Increment：X = X + 1，並比較 X 和 m
AND A = A AND m：位元 AND 運算
OR A = A OR m：位元 OR 運算

組譯器指令介紹

範例：ALPHA RESW 1

主要的組譯器指令

1. RESW（Reserve Word）

功能：保留指定數量的 word 空間（不初始化）

  • 用途：宣告變數、陣列
  • 不給初始值，記憶體內容未定義

2. RESB（Reserve Byte）

功能：保留指定數量的 byte 空間（不初始化）

3. WORD

功能：定義並初始化一個 word（3 bytes）

  • 與 RESW 的差別：WORD 會給初始值

4. BYTE

功能：定義並初始化一個或多個 byte

有兩種格式：

字元常數（Character constant）
十六進位常數（Hexadecimal constant）

5. START

功能：指定程式的起始位址

  • 必須是程式的第一行（除了註解）
  • 如果省略，預設從位址 0 開始

6. END

功能：標記程式結束，並指定第一個執行的指令

  • 必須是程式的最後一行
  • 運算元指定程式的進入點（entry point）

SIC/XE 介紹

SIC/XE = SIC with eXtra Equipment (或 eXtra Expense)
是 SIC 的增強版本，提供更大的記憶體、更多的暫存器和更強大的指令集

Memory

與 SIC 的差異

項目 SIC SIC/XE
記憶體大小 32,768 bytes (32 KB) 1,048,576 bytes (1 MB)
位址範圍 0 ~ 32,767 (2¹⁵) 0 ~ 1,048,575 (2²⁰)
位址欄位 15 bits 20 bits
  • 記憶體容量擴大 32 倍
  • 位址範圍：00000 ~ FFFFF（十六進位）

Register

SIC/XE 有 9 個暫存器（SIC 只有 5 個）

原有的 5 個暫存器（與 SIC 相同）

暫存器 編號 大小 用途
A 0 24 bits Accumulator（累加器）
X 1 24 bits Index register（索引暫存器）
L 2 24 bits Linkage register（連結暫存器）
PC 8 24 bits Program Counter（程式計數器）
SW 9 24 bits Status Word（狀態字組）

新增的 4 個暫存器

暫存器 編號 大小 用途
B 3 24 bits Base register（基底暫存器）
用於基底相對定址
S 4 24 bits General working register（通用工作暫存器）
T 5 24 bits General working register（通用工作暫存器）
F 6 48 bits Floating-point accumulator（浮點累加器）
唯一的 48 bits 暫存器

Data Format

1. 整數（Integer）

與 SIC 相同：

  • 有符號整數：-8,388,608 ~ 8,388,607 (-2²³ ~ 2²³-1)
  • 無符號整數：0 ~ 16,777,215 (0 ~ 2²⁴-1)

2. 浮點數（Floating-point）⭐ 新增

SIC/XE 新增了 48 bits 的浮點數格式：

  • s：符號位（0 = 正，1 = 負）
  • exponent：11 bits 指數（帶偏移量 1024）
  • fraction：36 bits 小數部分

Instruction Formats（指令格式）

SIC/XE 有 4 種指令格式（SIC 只有 1 種）

Format 1：1 byte（8 bits）

  • 不需要運算元的指令
  • 範例：FIX（浮點數轉整數）、FLOAT（整數轉浮點數）

Format 2：2 bytes（16 bits）

  • 暫存器對暫存器的操作
  • r1, r2：暫存器編號（0-9）
  • 範例：ADDR r1, r2（r2 = r2 + r1）

Format 3：3 bytes（24 bits）⭐ 最常用

nixbpe 旗標說明：

  • n：indirect bit（間接定址）
  • i：immediate bit（立即定址）
  • x：indexed bit（索引定址）
  • b：base relative bit（基底相對定址）
  • p：PC relative bit（PC相對定址）
  • e：extended format bit（擴展格式）

Format 4：4 bytes（32 bits）

  • Format 3 的擴展版本（e = 1）
  • 使用完整的 20 bits 位址
  • 可以直接定址整個 1 MB 記憶體
  • 範例：+LDA BUFFER（+ 表示 Format 4）

Addressing Modes（定址模式）

SIC/XE 有 6 種定址模式（SIC 只有 2 種）

定址模式判斷（透過 n, i, x, b, p 旗標）

Mode n i 說明 範例
1. 立即定址 0 1 Immediate：運算元是常數 LDA #5
2. 間接定址 1 0 Indirect：運算元是指標 LDA @RETADR
3. 簡單定址 1 1 Simple：一般定址 LDA ALPHA
4. 簡單定址 0 0 Simple：相容 SIC LDA ALPHA

目標位址計算

a. 立即定址（Immediate）：n=0, i=1

範例：LDA #5 → A = 5

b. 間接定址（Indirect）：n=1, i=0

範例：LDA @PTR → A = Memory[Memory[PTR]]

c. 簡單定址（Simple）：n=1, i=1 或 n=0, i=0

需要根據 b, p, e 計算目標位址：

b p e 計算方式
0 0 0 TA = disp（直接定址，少見）
0 1 0 TA = (PC) + disp（PC相對定址）
1 0 0 TA = (B) + disp（基底相對定址）
0 0 1 TA = address（直接定址，Format 4）

特別說明：

  • disp：12 bits 的位移量（-2048 ~ 2047，有符號）
  • address：20 bits 的完整位址（0 ~ 1048575）

d. 索引定址（Indexed）

如果 x = 1，在上述任何模式後再加上 (X)：

範例：LDA BUFFER,X

常見定址模式範例

重點整理

SIC/XE 的三大核心改進：

  1. 更大的記憶體：1 MB（支援大型程式）
  2. 更靈活的定址：立即定址、間接定址、相對定址（程式可重定位）
  3. 浮點運算：支援科學計算

實際應用場景

1. 簡單變數存取 → Simple

2. 常數載入 → Immediate

3. 陣列處理 → Indexed

4. 指標操作 → Indirect

5. 跨區段呼叫 → Extended

記憶技巧

ni 的組合

  • n=1, i=1 → Simple（正常定址）
  • n=0, i=1 → Immediate（#，立即值）
  • n=1, i=0 → Indirect（@，間接）
  • n=0, i=0 → SIC 相容模式

bp 的組合

  • b=0, p=1 → PC-relative
  • b=1, p=0 → Base-relative
  • b=0, p=0 → 直接定址（通常配合 e=1）

定址模式總結

  • PC-relative: TA = PC + disp
  • Base-relative: TA = B + disp
  • Indexed: TA = 計算位址 + X
